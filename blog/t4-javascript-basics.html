Explain the difference between accessing data from arrays and object literals

Arrays

Arrays are a list of values that have order. If you're accessing data from an array you will be referencing the the spot in the array where that value sits. Arrays also have methods that you can use in order to manipulate that order such as .push();, .pop();, .shift() and .unshift().

Objects

An object stores data as in two parts - a property and a value. Compared to arrays - object properties/values pairs can be stored in any order. If you are accessing data you will be accessing using the property name to access the value that is attached to that property. Because each value is assigned by a property name - object properties are easy to change. Changing array values is a little more risky as you have to find the value you want to change and reference that spot where the value is sitting to change it. If the array is huge... Well, things can get messy and you could very likely change the wrong array value.

Explain what functions are and why they are useful

Functions are chunks of code that perform a specific task. It's basically a mini programme that can be called on to do it's job when needed. Functions are super useful because you can give them information (values, parameters, arguments) and they will give you some information back. This all happens within the little world of the function and won't mess up the rest of your code. It also means that you can make many modular parts when you're coding and if one of them is not working - it won't screw with the ones that are.
