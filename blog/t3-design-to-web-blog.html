What is a responsive site, and why responsiveness is important

Everyone has a different sized computer, tablet, phone etc that they may be looking at your website from. Responsive website design means that a website will adjust itself to best fit the screen size that you aare looking at it in. This is important because it means that no matter how your website is being accessed the information will be presented in an easy to use an understand format.

What is mobile first design, and why it's important

Mobile first design is a design approach where a website is made for smaller screen sizes before larger. This is an important approach because it forces a website designer to organize the content of the website before focusing on beautifying the website.

What are frameworks, and their pros and cons

A framework (or boiler plate) is made up of code that you can utilize to build out the main structure of a website. We used one called Skeleton. By using premade tags, classes and snippets of code that will determine things like layout, form fields, typography and responsiveness you can quickly and efficiently build a website.

Pro's Con's
Responsive Can be limiting
Fast clean code building Can stop people from actually coding
Good Support from most framework owners Opensource code can be an easier for hackers to target

What a wireframe is and why we use it

A wireframe is a sketch of your website design that lays out basic elements of your website with a stronger focus on content/functionality over design.
